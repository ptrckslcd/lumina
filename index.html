About Lumina

What is Lumina?

Lumina bridges the gap between physical smart home hardware and interactive web experiences. It goes beyond a standard remote control by featuring a physics-simulated "Digital Twin" (our interactive web mascot) rendered on an HTML5 Canvas. This digital twin mirrors the physical ESP32 lamp's state, color, and effects in real-time, creating a seamless adaptive atmosphere.

How it Works

Lumina is engineered with a fault-tolerant, dual-communication bridge:

  • Local Edge Control (REST API): When on the same Wi-Fi network, the app communicates directly with the physical lamp's WLED JSON API. This provides zero-latency, high-speed data transfer for rapid color syncing and animations.
  • Global Cloud Control (MQTT): When the user leaves the local network, the app can be switched to a WebSockets-based MQTT protocol (via HiveMQ on port 8884). This allows the lamp to be controlled remotely from anywhere in the world.

Connect Your ESP32 Lamp

Quick setup path to pair your physical lamp with this web app:

  1. Flash WLED firmware: Install WLED to your ESP32 using install.wled.me.
  2. Wire hardware correctly: Connect LED data to your ESP32 data pin, use a stable 5V supply for LEDs, and make sure ESP32 ground and LED ground are shared.
  3. Join Wi-Fi: Configure WLED on your 2.4GHz network and note the device IP shown in WLED (example: http://192.168.1.120).
  4. Use REST for local control: In System tab, choose REST (Local), set WLED IP, then click Ping Target and Save Config.
  5. Use MQTT for remote control (optional): Set broker/topic in WLED Sync settings and mirror the same values in Lumina System tab under MQTT (Cloud).

Things to know: REST mode requires your dashboard and ESP32 to be on the same network. If local REST fails from a secure host, it is usually due to browser mixed-content/private-network restrictions. MQTT mode is the fallback for cross-network control.

Quick test: Open http://YOUR_WLED_IP/json/state in your browser. If it loads JSON, the lamp is reachable and Lumina can control it.

What to Do Next (App Basics)

After hardware connection, complete this quick first-run setup:

  1. OpenWeather API key: Go to OpenWeather API Keys, copy your key, then paste it in System → Weather Settings → OpenWeather API Key.
  2. Set default city: In System → Default Location, enter your city format (example: Nabua, PH), then click Save Config.
  3. Choose Active Mode: In System → Active Mode, select REST (Local) for same-network control, or MQTT (Cloud) for remote control.
  4. Verify connection: Click Ping Target. When status turns online, your app and lamp are linked.
  5. Run first weather sync: Go to Automations and click Sync Now to test weather-to-light mapping.

How Active Mode switching works:

  • REST (Local): Uses your WLED IP directly. Fastest response. Requires same local network.
  • MQTT (Cloud): Uses broker/topic values. Best for remote access or strict browser network policies.
  • Startup Default: Choose whether Lumina remembers last mode or forces REST/MQTT at boot.
  • When switching modes: Lumina reinitializes communication automatically. Use Ping to confirm active path.
Tab quick guide: Lighting = manual effects, Automations = weather/hydration behaviors, System = network and API config, Lab = testing and advanced payload overrides.

System Limitations & Security Constraints

  • Curated Effect Library: Lumina intentionally maps and supports only a specific subset of WLED effects to preserve its distinct visual atmosphere.
  • HTTPS Security (Mixed Content): Modern browsers enforce strict Private Network Access rules. Hosting Lumina on a secure https:// site (like Vercel or standard GitHub Pages) will block REST commands to local http://10.x.x.x WLED targets. Use local run, unencrypted http:// hosting (for example Surge), or MQTT mode.
  • Tab Throttling: Browsers (especially Chrome) throttle setInterval and requestAnimationFrame when a tab is minimized or in the background. Keep Lumina visible for accurate hydration timers and smooth canvas rendering.
  • Device Limit: The current architecture maps the Digital Twin to one WLED instance at a time.
  • Persistence: Settings are stored locally in your browser. Clearing site data/cache resets configured IP addresses, preferences, and API keys.

Tech Stack

  • Frontend: Vanilla JavaScript, HTML5 Canvas, CSS Variables
  • Hardware: ESP32 Microcontroller, WS2812B LEDs
  • Firmware: WLED JSON API
  • Networking: Paho MQTT, WebSockets, REST API
